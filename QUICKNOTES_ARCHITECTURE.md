# Архитектура и Документация модуля QuickNotes (Ежедневник)

Этот документ описывает текущую реализацию модуля QuickNotes для Foundry VTT. Он создан для того, чтобы другие разработчики (или нейросети) могли быстро понять структуру проекта, используемые технологии и реализованные функции.

## 1. Общая концепция
**QuickNotes** — это современный, быстрый и красивый блокнот для игроков и мастеров. Он использует стилистику «Glassmorphism» (полупрозрачность, размытие фона) и предлагает удобный интерфейс вкладок для различных типов заметок.

Ключевая особенность модуля: **разделение на личные и общие (Shared) рабочие пространства (доски).** 
- Личные записи хранятся в настройках конкретного пользователя (`game.user`).
- Общие записи (Доски) хранятся в специальных технических документах `JournalEntry` (например, "QuickNotes_Shared_DB").

## 2. Технический стек и API
- **Foundry API:** Модуль полностью написан на базе современного **ApplicationV2 API** (введено в Foundry V12).
- **Совместимость:** Обратная совместимость (насколько возможно) и поддержка грядущих версий (V13/V14) — устранены предупреждения о `TextEditor` и `FormDataExtended` (используются нативные браузерные `FormData`).
- **Стилизация:** Чистый CSS с использованием CSS-переменных (`var(--qn-accent)`), Flexbox и CSS Grid. Анимации реализованы через CSS `transition` и `scale`.
- **Шаблонизатор:** Handlebars (`.hbs`).

## 3. Структура файлов
- `module.json` — Манифест модуля.
- `scripts/main.js` — Точка входа. Содержит хуки инициализации (`init`, `ready`, `renderSceneControls`) и логику плавающего виджета (Floating Action Button) на экране.
- `scripts/app.js` — Основной класс приложения `QuickNotesApp`, наследуемый от `foundry.applications.api.ApplicationV2`. Содержит всю логику работы с данными, рендеринга, drag-n-drop и бесконечной доски.
- `styles/style.css` — Стили модуля. Разделен на логические блоки: окно, вкладки, карточки, доска, виджет.
- `templates/content.hbs` — Главный шаблон содержимого вкладок и доски.
- `templates/tabs.hbs` — Шаблон навигационного меню вкладок.

## 4. База данных (Флаги)
Данные сохраняются через механизм флагов Foundry (`setFlag` / `unsetFlag`).
Структура объекта `data`:
```json
{
  "notes": { "id123": { "text": "...", "color": "yellow", "sort": 0, "onBoard": false } },
  "npc": { "id456": { "name": "...", "location": "...", "attitude": "...", "note": "...", "sort": 1 } },
  "quests": { "id789": { "text": "...", "status": "active" } },
  "timeline": { "id000": { "time": "...", "event": "..." } },
  "links": [ { "source": "id123", "target": "id456", "label": "Враги" } ]
}
```

## 5. Основной функционал (Уже реализовано)

### 5.1. Плавающий виджет и Быстрое создание
- Иконка книжки, которую можно свободно перемещать по экрану (сохраняет позицию).
- При наведении (Hover) всплывает полукруглое меню (Radial Menu) для быстрого создания записей.
- Задержка скрытия меню (1.5 сек) предотвращает случайное закрытие ("дергание").
- Данные из меню быстрого создания сохраняются в базу с автоматическим вычислением правильного порядкового номера (`sort: maxSort + 1`).

### 5.2. Вкладки и Карточки
- **Заметки, NPC, Квесты, Хронология:** 4 типа карточек. Каждая карточка стилизуется под "стикер" выбранного цвета.
- **Живая сортировка (Live Drag & Drop):** Реализован нативный drag-n-drop списка. При перетаскивании элементы динамически (через `insertBefore`) меняются местами в DOM. По завершении перетаскивания новые индексы сохраняются в базу.
- **Режим редактирования (Inline editing):** По клику на кнопку редактирования текст карточки превращается в поля ввода/textarea. Сохранение происходит по клику вне элемента.

### 5.3. Бесконечная Доска (Board View)
- Карточки можно "закрепить на доске" (onBoard = true).
- **Интерактивность:** Доска поддерживает панорамирование (Pan) и зум (Zoom).
- **Drag & Drop на доске:** Карточки позиционируются абсолютно (через координаты `boardX` и `boardY`).
- **Связи (Links):** Между карточками можно натягивать нити (создаются SVG-линии). Линиям можно задавать текстовые ярлыки (labels). Удаление связи — правый клик по ярлыку.

### 5.4. Рабочие пространства (Workspaces)
- Вкладка настроек позволяет создать новое рабочее пространство (создается `JournalEntry` с правами Observer/Owner для нужных игроков).
- Переключение между пространствами моментально перезагружает контекст приложения.

### 5.5. Глобальный поиск
- Вкладка поиска итеративно проходит по всем данным во всех вкладках (Notes, NPC, Quests, Timeline) и выводит совпадения.

### 5.6. Настройки и Персонализация
- Игрок может настраивать прозрачность фона, акцентный цвет (применяется ко всем элементам интерфейса через CSS-переменные).
- Доступен импорт и экспорт текущего пространства в формате JSON.

## 6. Идеи для будущих улучшений
Ниже перечислены идеи, которые можно реализовать в следующих обновлениях:
1. **Генерация контента (AI):** Интеграция генерации описаний NPC или идей для квестов прямо в карточках.
2. **Папки или Теги:** Добавление системы тегов для карточек, чтобы их можно было фильтровать внутри одной вкладки.
3. **Rich Text Editor:** Использование полноформатного редактора (ProseMirror / TinyMCE) для поля заметок вместо обычного `<textarea>`.
4. **Мини-карта для доски:** Создание окошка навигации, показывающего, где находятся карточки на огромной доске.
5. **Мультиплеер-курсоры:** Отображение курсоров других игроков в режиме реального времени на общей доске.
